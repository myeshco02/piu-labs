Uwagi

Ogólne uwagi do formularzy

Fundament interaktywnego internetu

Formularze rónego typu stały się punktem zwrotnym dla rozwoju internetu - wcześniej użytkownik był biernym odbiorcą statycznych treści. Formularze umożliwiły kontrolowaną ale i rozbudowaną formę komunikacji z serwerem.

Uniwersalność

Formularze można stosować wszędzie tam, gdzie chcemy uzyskać jakikolwiek input od użytkownika

  • rejestracja i logowanie
  • newslettery, subsktypcje, marketing
  • ankiety, zbieranie surowych danych, zapisy, bookowanie
  • sklepy internetowe, płatności, bankowość
  • komentarze, fora, platformy społecznościowe
  • konfiguratory, kreatory
  • współczesne aplikacje webowe

Kiedy stosować

Praktycznie wszędzie w jakieś formie. W dzisiejszym internecie praktycznie nie ma już stron, które nie mają żadnego formularza (najprostsze logowanie, komentarz, newsletter, kontakt = jakiś rodzaj formularza)

Kiedy nie stosować

Nie są wymagane dla czysto statycznych, informacyjnych stron typu prosta dokumentacja markdown (np. czysty docosaurus bez wyszukiwania)

W praktyce pytanie można zadać inaczej: Kiedy nie stosować 'czystych HTML-owych formularzy?

Odpowiedź brzmi: wszędzie gdzie potrzeba reponsywnego UX, interaktywności, dynamicznej walidacji po stronie frontend -> sięgamy po JS/inne frameworki

Dodatkowo

CSS i HTML

O ile HTML jest dość czytelny, odpowiada za strukturę, to CSS można łatwo przeładować i się pogubić. Dziejszy CSS3 jest jednak dość potężny jak chodzi o animacje itp. Jednak lepiej działać z nim modularnie, żeby nie skończyć z jednym plikiem z tysiącami linijek. Po CSS dobrze jednak poznać inne frameworki typu Tailwind. W czystym CSS panowanie nad spójnością designu jest skomplikowane i okazuje się, że w szczególnych przypadkach How to center a divwydaje się niemożliwe. Nie mówiąc o tym, że różne przeglądarki lubią rysować strony w różny sposób...

Formularz jako podstawa

Klasyczne formularze HTML to solidna podstawa praktycznie każdej strony. Zapewniają interaktywność dla użytkowników.

Bez JS - da się, ALE...

W dzisiejszych czasach JS lub inne pokrewne frameworki w zakresie tematu formularzy zdecydowanie poszerzają horyzonty w zakresie UI/UX
